IoT-Enabled Welding Robot Cell (Smart Controller)

Android HMI → Smart Controller → Robot | Modbus-TCP bridge with push-button fallback & safety interlocks.

What

  • Replaced the traditional PLC with a smart controller–based master–slave Modbus-TCP/IP controller, serving as the central control unit of the welding robot cell.
  • Developed the complete solution architecture covering register mapping, command logic, real-time robot monitoring, and safety logic for reliable operation.
  • Built an Android HMI interface for robot program execution, servo control, alarm reset, and production count tracking, enabling IoT-style supervisory control.

How

  • Configured the smart controller to act as a Modbus-TCP client to the robot and a Modbus-TCP server to the Android HMI, forming a reliable master–slave bridge that enables full bidirectional communication.
  • Ensured continuous robot cell operation even if Wi-Fi or the Android tablet disconnects, by keeping the smart controller persistently connected to the robot and allowing control through physical push-buttons.
  • Created systemd service files on the smart controller to automatically start the Ethernet, Wi-Fi, and Python control script during boot-up, with proper dependency ordering and auto-restart policies for reliability.

Results

  • 75% reduction in control system material cost by replacing the PLC/IPC layer with the smart controller.
  • Improved cycle stability and responsiveness through persistent Modbus-TCP communication and local execution on the smart controller.
  • Robust, Wi-Fi-independent architecture that maintains full robot operation through physical push-buttons even during Android disconnects.
  • Flexible, service-oriented controller that can be extended for additional cells or future integrations.
